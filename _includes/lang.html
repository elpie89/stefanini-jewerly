{%- comment -%} Language-aware variables for templates. NOTE: jekyll-multiple-languages-plugin sets site.baseurl = "/" for non-default languages and exposes the original baseurl as site.baseurl_root. So `relative_url` already produces the correctly-prefixed URL within the current language — DO NOT prepend lang_prefix manually before `relative_url`. Vars exported: active — current language code default_lang — site default t — site.data.i18n[active] translation tree other_lang — the other language code other_url — fully-qualified path to the home of the other language (use directly as href — do NOT pipe through relative_url) {%- endcomment -%} {%- assign active = site.lang | default: site.default_lang -%} {%- assign default_lang = site.default_lang | default: site.lang -%} {%- assign t = site.data.i18n[active] -%} {%- assign other_lang = "" -%} {%- for l in site.languages -%}{%- if l != active -%}{%- assign other_lang = l -%}{%- endif -%}{%- endfor -%} {%- assign baseurl_root = site.baseurl_root | default: site.baseurl -%} {%- if other_lang == default_lang -%} {%- assign other_url = baseurl_root | append: "/" -%} {%- else -%} {%- assign other_url = baseurl_root | append: "/" | append: other_lang | append: "/" -%} {%- endif -%}
